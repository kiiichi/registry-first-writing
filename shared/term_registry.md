# Term Registry

Use this file to stabilize high-frequency or easy-to-drift terminology.

This file is for current preferred wording, not historical wording experiments.
Keep only active project truth and a small number of reserved starter slots here.
For format examples, see `shared/references/term_registry_examples.md`.

## Entry Guide

Required fields:
- `term_id`
- `concept`
- `preferred_term`
- `definition_notes`
- `status`

Optional fields:
- `abbreviation`
- `discouraged_variants`
- `first_definition_target_location`
- `notes`

Allowed `status` values:
- `proposed`
- `confirmed`
- `deprecated`
- `reserved`
