# Claim Register

Use this file to freeze the central claims, contribution wording, scope boundaries, and claim-strength limits of the project.

This file is a current-truth registry, not a patch log.
Keep only active project truth and a small number of reserved starter slots here.
For format examples, see `shared/references/claim_register_examples.md`.

## Entry Guide

Required fields:
- `claim_id`
- `status`
- `claim_text`
- `scope_notes`
- `evidence_basis`
- `target_locations`

Optional fields:
- `quantitative_anchors`
- `forbidden_drift`
- `notes`

Allowed `status` values:
- `proposed`
- `confirmed`
- `deprecated`
- `reserved`
