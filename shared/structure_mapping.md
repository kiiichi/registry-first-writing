# Structure Mapping

Use this file to stabilize document structure, section roles, content ownership, and evidence placement.

This file should answer two questions:
1. what each section is for,
2. what each section must not be overloaded with.

Keep only active project truth and a small number of reserved starter slots here.
For format examples, see `shared/references/structure_mapping_examples.md`.

## Entry Guide

Required fields:
- `section_id`
- `section_name`
- `intended_function`
- `what_belongs_here`
- `what_should_not_be_overloaded_here`
- `relation_to_major_claims`
- `evidence_detail_placement_notes`
- `status`

Optional fields:
- `notes`

Allowed `status` values:
- `proposed`
- `confirmed`
- `deprecated`
- `reserved`
